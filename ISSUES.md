# Инструкция для доработок

Приступая к исправлениям, следуй этому процессу:

1. **Выбери задачу**, начиная с критических.
2. **Активируй соответствующего агента**:
   - Для UI/UX: `frontend-developer` или `ui-designer`
   - Для бизнес-логики: `senior-developer`
   - Для тестов: `api-tester` или `test-results-analyzer`
3. **Внеси изменения**, убедись, что исправление затрагивает только указанную проблему.
4. **После правки обязательно проверь**:
   - Запусти тесты, если они есть.
   - Если тестов нет – напиши минимальный тест для проверяемого функционала.
   - Проверь вручную через браузер (если требуется).
5. **Обнови ISSUES.md** – отметь задачу как выполненную (`[x]`).
6. **Переходи к следующей задаче**.

Не пытайся исправлять несколько проблем в одном запросе – это приводит к неполным правкам.

# Замечания по проекту

## UI/UX
- [ ] **Левая панель содержит навигационное меню вместо иерархии каталога**
  *Проблема:* В левой части страницы отображается меню («Каталог», «Заказы», «Склад» и т.п.), которое дублирует выпадающее меню пользователя.
  *Ожидание:* Левая панель должна содержать **иерархическое дерево категорий запчастей** только на странице каталога. В других разделах должно быть обычное меню или без него.
  *Приоритет:* критично

- [ ] **В центральной области не отображаются запчасти для выбранного узла**
  *Проблема:* При клике на узел в дереве центральная область не загружает список запчастей.
  *Ожидание:* При выборе узла в дереве должен отправляться запрос к API (`/api/parts?nodeId=...`), а ответ должен отображаться в центральной области (плиткой или таблицей).
  *Приоритет:* критично

- [ ] **Дерево категорий не работает**
  *Проблема:* Ошибка "Maximum update depth exceeded". Дерево отображается во всех разделах (админка, заказы), а должно быть только в каталоге.
  *Ожидание:* HierarchyTree должен быть только в `/catalog`. В других разделах левая панель не нужна или должна быть другой.
  *Приоритет:* критично

- [ ] **Хлебные крошки отсутствуют или не синхронизированы**
  *Проблема:* Нет хлебных крошек, показывающих путь к текущему узлу.
  *Ожидание:* Верхняя панель должна содержать хлебные крошки, построенные на основе `path` выбранного узла.
  *Приоритет:* важно

- [x] **Header**
  *Проблема:* Header был доступен только на главной странице.
  *Решение:* AppLayout с Header перемещён в корневой layout ([locale]/layout.tsx). Теперь Header отображается на всех страницах приложения.
  *Приоритет:* критично ✅ ИСПРАВЛЕНО

- [ ] **Нужно доработать Header**
  *Проблема:* Выбор языка кнопками.
  *Ожидание:* Выбор языка должен производиться в выпадающем меню. При выборе арабского языка, элементы Header не должны смещаться влево.
  *Приоритет:* важно

- [ ] **Мультиязычность**
  *Проблема:* Перевод не работает почти нигде. Вместо слов пишется catalog.title, catalog.items и т.д., вне зависимости от выбранного языка.
  *Ожидание:* Правильная мультиязычность для всего проекта. Нужно проверить и обновить локали.
  *Приоритет:* важно
